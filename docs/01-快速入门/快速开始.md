# 快速开始

本文档帮助你在 5 分钟内启动和运行项目。

## 环境要求

| 依赖 | 版本 | 下载链接 |
|------|------|----------|
| Go | 1.24+ | [下载](https://go.dev/dl/) |
| Podman | 5.0+ | [安装](https://podman.io/getting-started/installation) |

## 启动（podman kube play 一键全栈）

```bash
# 1. 克隆项目
git clone https://github.com/masonsxu/cloudwego-microservice-demo.git
cd cloudwego-microservice-demo

# 2. 准备配置（修改 Secret 中四个 CHANGE_ME_* 占位密码）
cp docker/pod.yml.example docker/pod.yml
vim docker/pod.yml

# 3. 构建 Go 服务镜像（identity-srv:latest 和 gateway:latest）
./scripts/build-images.sh

# 4. 一键启动全栈（5 基础设施 + identity_srv + gateway，共 7 个 Pod）
podman kube play docker/pod.yml

# 5. 验证服务
curl http://localhost:8080/ping
# 预期返回: {"message":"pong"}
```

### 服务访问入口

| 服务 | 地址 |
|------|------|
| API 网关 | http://localhost:8080 |
| Swagger 文档 | http://localhost:8080/swagger/index.html |
| 健康检查 | http://localhost:8080/ping |
| Jaeger 链路追踪 | http://localhost:16686 |

### 常用命令

```bash
podman kube play docker/pod.yml                       # 启动全栈
podman kube play --down docker/pod.yml                # 停止全栈（保留数据卷）
podman kube play --down --force docker/pod.yml        # 停止并清空数据卷
podman pod ps                                         # 查看 Pod 状态
podman ps                                             # 查看容器状态
podman logs -f gateway-pod-gateway                    # 查看 gateway 日志
podman logs -f identity-srv-pod-identity-srv          # 查看 identity_srv 日志
podman logs -f postgres-pod-postgres                  # 查看 postgres 日志
```

### 修改代码后重新部署

```bash
./scripts/build-images.sh gateway          # 或 identity 或不带参数全部构建
podman kube play --down docker/pod.yml
podman kube play docker/pod.yml
```

## API 示例

### 用户登录

```bash
curl -X POST http://localhost:8080/api/v1/identity/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "password123"
  }'
```

响应示例：

```json
{
    "base_resp": {
        "code": 0,
        "message": "success"
    },
    "data": {
        "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "token_type": "Bearer",
        "expires_in": 1800
    }
}
```

### 获取用户信息

```bash
curl -X GET http://localhost:8080/api/v1/identity/users/me \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```
