# Workbench 配色规范（精简版）

> 这份文档面向**产品、UI 评审、非工程**同事。**完整设计 token、组件规范、暗色对偶**请看仓库根目录 [`DESIGN.md`](../../DESIGN.md)。
>
> 一句话原则：**让数据成为主角，UI 是它的骨架。**

---

## 一、唯一交互色

整个站点中所有"可点"的元素只用**一个**颜色：

| 名称 | Hex | 何处使用 |
|---|---|---|
| **深靛蓝 / Indigo 600** | `#4F46E5` | 主按钮、链接、Focus 环、Sidebar 激活态左侧 2px、Switch 开启、Checkbox 勾选 |

**配套**：

| Token | Hex | 用途 |
|---|---|---|
| `primary-hover` | `#4338CA` | 主按钮 hover |
| `primary-active` | `#3730A3` | 主按钮按下；徽标 primary 文字色 |
| `primary-soft` | `#EEF2FF` | Sidebar 激活项底；表格选中行；徽标 primary 底 |

> **铁律**：**不引入第二个交互色**。哪怕"图标想丰富一些"——丰富的代价是失去识别度。

---

## 二、表面色（三档梯队）

白色底之间的 1–4% 色差替代了大部分阴影，让层级靠"色温"而不是"投影"。

| Token | Hex | 用在哪 |
|---|---|---|
| `canvas` | `#FFFFFF` | 主内容区底；卡片底；Topbar；Dialog 内层 |
| `sunken` | `#F5F5F7` | Sidebar 底；表格 hover 行；输入 disabled 底 |
| `overlay` | `#FFFFFF` | dropdown / popover / dialog 内容层（与 canvas 等值，但语义独立） |

```
┌────────┬──────────────────────────────────────┐
│ Sidebar│  Topbar  (canvas, border-bottom)     │
│ (sunken)├──────────────────────────────────────┤
│        │                                       │
│        │   Content (canvas)                    │
│        │     ┌── card (canvas + border) ──┐   │
│        │     │ Hover row → bg sunken      │   │
│        │     └────────────────────────────┘   │
└────────┴──────────────────────────────────────┘
```

## 三、文字色

| Token | Hex | 用途 |
|---|---|---|
| `ink-strong` | `#0A0A0B` | 页面主标题（display 级别） |
| `ink` | `#1F2024` | 默认正文、表格单元、表单值 |
| `ink-muted` | `#52525B` | 次级说明、表头、描述文字 |
| `ink-subtle` | `#9CA3AF` | 占位符、面包屑分隔符 |
| `ink-disabled` | `#D4D4D8` | disabled 文字 |

> 不用纯黑（#000）。`#0A0A0B` 是带 1% 蓝调的近黑，与白配对时观感比纯黑更"光学正确"。

---

## 四、语义色（仅描述状态，不参与"操作意图"）

| 语义 | 实色 | 软底 | 文字色 | 用途 |
|---|---|---|---|---|
| Success | `#10B981` | `#ECFDF5` | `#047857` | 操作成功；用户激活；服务正常 |
| Warning | `#F59E0B` | `#FFFBEB` | `#B45309` | 配额预警；待审批；非破坏性提示 |
| Danger | `#EF4444` | `#FEF2F2` | `#B91C1C` | 删除按钮；锁定/禁用；表单校验错误 |
| Info | `#3B82F6` | `#EFF6FF` | `#1D4ED8` | 中性通知；新功能横幅 |

> **不能把 success / warning 当主色用**——它们只描述状态，不描述"操作意图"。
> 例如：不要把"导出"按钮做成绿色"成功色"——它是一次操作，不是一种状态。

## 五、图表色板（多系列数据可视化）

| 系列 | 亮色 Hex | 暗色 Hex | 推荐用法 |
|---|---|---|---|
| chart-1 | `#4F46E5` | `#818CF8` | **主系列**（与品牌主色一致，强化"主数据"识别） |
| chart-2 | `#0EA5E9` | `#38BDF8` | 第二系列（指标 vs 对比基线） |
| chart-3 | `#10B981` | `#34D399` | 第三系列（绿，可暗示"达成 / 健康"） |
| chart-4 | `#F59E0B` | `#FBBF24` | 第四系列（黄，可暗示"预警"） |
| chart-5 | `#EC4899` | `#F472B6` | 第五系列（粉，纯视觉区分） |
| chart-6 | `#8B5CF6` | `#A78BFA` | 第六系列（紫，纯视觉区分） |

**规则**：
- 单系列 → 永远 `chart-1`
- 多系列 → 按 `chart-1 → chart-6` 顺序，不跳号
- 超过 6 系列 → 先尝试合并次要项（"其他 N 项"），不引入第 7 个色

## 六、主题（亮色 / 暗色）

- **默认亮色**——管理后台业界主流（Linear / Vercel / GitHub / Notion），长时间表格阅读不疲劳
- **暗色对偶**——所有 token 一一映射；切换通过顶栏右上角 ☀️/🌙 完成
- **不要为暗色单独再设计页面**：用同一套语义 token，让浏览器切换变量即可

---

## 七、字体

| 角色 | 字体栈 |
|---|---|
| 正文 / 标题 | `"PingFang SC", "Source Han Sans CN", "Noto Sans CJK SC", system-ui, "Segoe UI", "Microsoft YaHei", sans-serif` |
| 代码 / 数字 / API 端点 | `"JetBrains Mono", "SF Mono", "Cascadia Code", Consolas, monospace` |

- **正文 14px**（不是 16/17px）——后台密度优先
- **数字列用 `tabular-nums`** 让数字对齐（表格、价格、计数全部用）

---

## 八、Do / Don't（七条铁律）

### ✅ Do
1. 任何"可点"元素只用 **#4F46E5**
2. Focus 用 **3px 半透明 ring**，不要用粗边框
3. 表格 hover 行底色用 **sunken**，不要加左侧色条
4. 状态徽标用语义变体（`active=success / inactive=default / suspended=warning / locked=destructive`）
5. 数字一律 **tabular** + 右对齐
6. 暗色主题切换通过 `:root[data-theme]` 覆盖语义 token
7. 每加一个组件，**立刻在暗色下检查一遍**

### ❌ Don't
1. 不引入第二个交互色（绿按钮 / 橙链接 / 金色 hover）
2. 不给按钮、卡片、表格行、徽标加阴影（阴影只属于 dropdown / popover / dialog）
3. 不在背景挂渐变 / 网格点 / 模糊光斑 / shimmer 装饰动画
4. 不硬编码字体（`font-['Inter']`）
5. 不混用圆角语法（按钮全 6px，卡片全 8px，dialog 全 12px）
6. 不在 Sidebar 高亮态用大面积底色（左侧 2px primary 边线 + 白底就够）
7. 不用纯黑（#000）做文字——用 `#0A0A0B`

## 九、视觉评审检查表

UI 评审时可对照下表逐项过：

| 检查项 | 方式 |
|---|---|
| 配色 | 全站可点元素是否只有 #4F46E5？语义色是否只在状态徽标 / 提示出现？ |
| 字体 | 表格、表单、按钮是否都是 14px？标题是否走 display-md/lg？数字是否 tabular？ |
| 间距 | 卡片内 20px，区块间 24px，没有 80px 大留白？ |
| 阴影 | 默认元素是否 elevation = none？只有 dropdown / popover / dialog 有阴影？ |
| 圆角 | 按钮 6px / 卡片 8px / dialog 12px / 徽标 4px / 圆点 9999？ |
| 暗色 | 切换暗色后所有色对仍可读，无残色翻转？ |
| 中英 | 切换中英文，布局不被长字符串撑破？ |
