## API 客户端管理

### ⚠️ 重要：自动生成代码的 Token 配置问题

**问题根因**：`openapi-typescript-codegen` 生成的代码默认 `TOKEN: undefined`，导致认证失败。

**解决方案**：使用配置文件集中管理，避免手动修改生成的文件。

### 重新生成 API 客户端

```bash
cd web
npm run regen-api
```

此脚本会：
1. 下载最新的 Swagger 文档
2. 重新生成 API 客户端代码
3. **注意**：TOKEN 配置由 `src/api/config.ts` 在应用启动时自动设置，无需手动修改

### API 客户端配置

在 `src/api/config.ts` 中集中管理：

```typescript
import { OpenAPI } from '@/api/generated'

// 应用启动时调用
export function initApiClient() {
  OpenAPI.TOKEN = () => Promise.resolve(localStorage.getItem('token') || '')
  OpenAPI.BASE = 'http://192.168.20.66:8088'
}

// 用户登出时调用
export function clearApiClient() {
  OpenAPI.TOKEN = undefined
}
```

### 使用生成的 API

```typescript
import { roleApi } from '@/api/role'

// ✅ 使用生成的 API（自动带 token）
const response = await roleApi.getRoles({ name: 'admin' })
```

### 何时使用自定义 request.ts

仅在以下情况使用 `src/api/request.ts`：
- 需要特殊的请求/响应拦截逻辑
- 需要处理生成代码不支持的场景
- 处理文件上传/下载等特殊场景

## 样式规范

### CSS @import 顺序规则

**规则**：在 CSS/PostCSS 文件中，所有 `@import` 语句必须位于文件最顶部，在任何其他语句（`@theme`、`@layer`、规则集等）之前。

```css
/* ✅ 正确 */
@import "tailwindcss";
@import './element-theme.scss';

@theme { ... }

body { ... }
```

```css
/* ❌ 错误：@import 在 @theme 之后，PostCSS 会忽略它 */
@import "tailwindcss";

@theme { ... }

@import './element-theme.scss';   /* 此 import 会被静默忽略！ */
```

**后果**：违反此规则时，PostCSS 会报 `@import must precede all other statements` 错误，并静默忽略该 import，导致被引入的样式文件完全不生效，且不易察觉（页面不报运行时错误）。

**适用范围**：`tailwind.css` 及项目中所有 `.css` 文件。

### 样式文件职责划分

| 文件 | 职责 |
|------|------|
| `variables.css` | 自定义设计系统变量（`--bg-*`、`--c-*`、`--shadow-*` 等） |
| `element-theme.scss` | El+ CSS 变量覆盖（`:root` 块） + 组件级选择器覆盖（`.el-table` 等） |
| `tailwind.css` | Tailwind 主题配置（`@theme`）+ 全局样式 + 工具类 |

每类变量只在一处维护，避免跨文件重复声明。

### 国际化 i18n Key 使用规范

**规则**：只能将**叶子节点 key** 传给 `t()` 函数用于渲染文本；将嵌套对象 key 传给 `t()` 时，vue-i18n v9 会返回 key 路径字符串而非翻译文本。

```typescript
// zh-CN.ts
user: {
  status: {          // ← 嵌套对象，非叶子节点
    active: '活跃',  // ← 叶子节点 ✅
    locked: '锁定',  // ← 叶子节点 ✅
  }
}
```

```html
<!-- ✅ 正确：使用叶子节点 key -->
<el-table-column :label="t('common.status')" />
<span>{{ t('user.status.active') }}</span>

<!-- ❌ 错误：user.status 是对象，渲染结果为 "USER.STATUS"（经 text-transform 后） -->
<el-table-column :label="t('user.status')" />
```

**检查方式**：凡是在 `locales/` 中定义了子键的 key（如 `user.gender`、`user.status`、`role.permissionLevel`），一律不能直接作为显示文本使用。



### 组件开发

1. **使用 Composition API**

   - 在 `src/locales/` 中添加翻译

## 页面加载状态规范

### 设计原则

页面切换时 **禁止出现白屏闪烁**。用户导航到新页面后，必须在首帧即看到页面骨架或加载占位，而非空白内容区。

### 页面类型与加载策略

| 页面类型 | 加载策略 | 组件 | 原因 |
|---------|---------|------|------|
| 列表页 | 骨架屏 | `ListPageSkeleton` | el-table 初始化需要 1-2 帧计算列宽，v-loading 会闪烁空表格结构 |
| 详情页 | 骨架屏 | `DetailPageSkeleton` | 内容卡片依赖后端数据，v-loading 覆盖空白卡片视觉效果差 |
| 表单页 | `v-loading` 遮罩 | — | 表单结构始终在 DOM 中（输入框有默认值），遮罩即可 |
| Dashboard | `v-loading` 遮罩 | — | 轻量 DOM，无 el-table，遮罩足够 |

### 双状态模式（initialLoading + loading）

所有需要骨架屏的页面 **必须** 使用双状态模式：

```typescript
const initialLoading = ref(true)   // 首次加载 → 骨架屏 vs 真实内容
const loading = ref(false)          // 后续刷新 → v-loading 遮罩覆盖已有内容
```

- `initialLoading`：控制 `v-if/v-else` 在骨架屏与真实内容之间切换，仅在 `onMounted` 的 `finally` 中设为 `false`
- `loading`：控制 `v-loading` 指令，在每次数据请求时 `true/false` 交替，覆盖已渲染的内容

**为什么需要双状态**：
- 仅用 `loading`：首次加载结束后若触发刷新（编辑保存、删除后重新拉取），`loading=true` 会再次显示骨架屏，导致已有内容消失
- 仅用 `initialLoading`：后续刷新操作无加载反馈

### 列表页规范

**页面结构**：页头、统计卡片、搜索栏始终在 DOM 中，骨架屏 **仅替换表格区域内部内容**。

```vue
<template>
  <div class="xxx-list">
    <!-- 始终渲染 —— 页头、统计、搜索 -->
    <div class="page-header">...</div>
    <div class="stats-row">...</div>
    <div class="search-section">...</div>

    <!-- 表格容器始终渲染（保持边框和圆角），内部切换 -->
    <div class="table-card">
      <ListPageSkeleton v-if="initialLoading" :columns="6" :rows="8" />
      <template v-else>
        <el-table v-loading="loading" :data="list">...</el-table>
        <div class="pagination-wrapper">...</div>
      </template>
    </div>
  </div>
</template>

<script setup lang="ts">
const initialLoading = ref(true)
const loading = ref(false)

const loadData = async () => {
  loading.value = true
  try {
    // fetch data...
  } finally {
    loading.value = false
    initialLoading.value = false   // 首次加载后永不再变为 true
  }
}

onMounted(() => { loadData() })
</script>
```

**关键约束**：
- 禁止用 `v-if/v-else` 包裹整个页面（会导致 el-table 初始化闪烁）
- `ListPageSkeleton` 的 `:columns` 和 `:rows` 应与实际表格列数和预期行数匹配
- `initialLoading.value = false` 必须在 `finally` 块中，保证即使请求失败也能切换到真实内容

### 详情页规范

**页面结构**：页头（`el-page-header`）始终可见，骨架屏替换整个内容区域。

```vue
<template>
  <div class="xxx-detail">
    <!-- 始终渲染 —— 页面头部和操作按钮 -->
    <el-page-header @back="handleBack">...</el-page-header>

    <!-- 内容区域切换 -->
    <div class="detail-content">
      <DetailPageSkeleton
        v-if="initialLoading"
        :side-span="8"
        :main-span="16"
        :side-cards="1"
        :main-cards="2"
        :show-avatar="true"
        :side-item-counts="[4]"
        :main-item-counts="[6, 4]"
      />
      <el-row v-else v-loading="loading" :gutter="20">
        <!-- 真实内容 -->
      </el-row>
    </div>
  </div>
</template>

<script setup lang="ts">
const initialLoading = ref(true)
const loading = ref(false)

async function fetchData() {
  loading.value = true
  try { /* ... */ }
  finally { loading.value = false }
}

onMounted(async () => {
  try {
    await Promise.all([fetchData(), fetchOtherData()])
  } finally {
    initialLoading.value = false
  }
})
</script>
```

**DetailPageSkeleton Props 说明**：

| Prop | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `sideSpan` | number | 8 | 左列栅格宽度（Element Plus 24 栅格） |
| `mainSpan` | number | 16 | 右列栅格宽度 |
| `sideCards` | number | 1 | 左列卡片骨架数量 |
| `mainCards` | number | 2 | 右列卡片骨架数量 |
| `showAvatar` | boolean | true | 左列首张卡片是否显示头像占位 |
| `sideItemCounts` | number[] | [4] | 左列每张卡片的描述条目数 |
| `mainItemCounts` | number[] | [6, 4] | 右列每张卡片的描述条目数 |

### 表单页 / Dashboard 规范（v-loading 方式）

```typescript
const loading = ref(true)   // 初始值必须为 true，确保首帧即显示遮罩

onMounted(async () => {
  try {
    await Promise.all([fetchA(), fetchB()])   // 多个请求必须用 Promise.all 同步
  } finally {
    loading.value = false
  }
})
```

### 多异步请求同步

当 `onMounted` 中有多个并行异步请求时，**必须** 使用 `Promise.all` 统一等待：

```typescript
// ✅ 正确：所有请求完成后统一关闭 loading
onMounted(async () => {
  try {
    await Promise.all([fetchData(), fetchTree(), fetchUsers()])
  } finally {
    initialLoading.value = false
  }
})

// ❌ 错误：各请求独立调用，loading 由最先返回的请求关闭
onMounted(() => {
  fetchData()       // 内部 loading.value = false 可能在 fetchTree 完成前触发
  fetchTree()
  fetchUsers()
})
```

### 禁止的做法

| 做法 | 问题 |
|------|------|
| `loading = ref(false)` 作为首次加载状态 | 组件挂载到 loading=true 之间有间隙，产生白屏 |
| `v-loading` 覆盖整个页面根 div（含页面头部） | 页头在加载期间不可见，无导航反馈 |
| `v-if/v-else` 包裹列表页整个模板 | el-table 从 DOM 移除再插入时需 1-2 帧初始化，产生空表格闪烁 |
| 多个并行请求各自独立控制 loading | 最先返回的请求提前关闭加载状态，其他数据区仍为空 |
| 骨架屏内使用 `el-card` / `el-table` 等重量级组件 | 骨架屏本身应纯 CSS 实现，避免组件初始化开销 |

### 骨架屏组件速查

| 组件 | 路径 | 适用场景 |
|------|------|---------|
| `ListPageSkeleton` | `src/components/skeleton/ListPageSkeleton.vue` | 列表页表格区域骨架 |
| `DetailPageSkeleton` | `src/components/skeleton/DetailPageSkeleton.vue` | 详情页两栏卡片骨架 |

骨架屏使用金色微光动画（`rgba(212, 175, 55, 0.04) → 0.10`），与项目深色主题配色一致。

## 常见问题

### Q: 重新生成 API 后认证失败？
A: 确保在 `App.vue` 的 `onMounted` 中调用了 `initApiClient()`

### Q: 生成的类型不符合需求？
A: 不要修改生成的文件，在 `src/types/` 中扩展类型

### Q: 如何添加新的 API 端点？
A:
1. 后端先修改 IDL 并重新生成
2. 前端运行 `npm run regen-api`
3. 在对应的 API 文件（如 `role.ts`）中封装调用方法
